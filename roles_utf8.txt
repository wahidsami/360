QA Role Detailed Specifications</h1><h2>Purpose</h2><p>Quality Assurance Engineer focused on testing, bug reporting, and verification workflows.</p><h2>Landing Page</h2><p><strong>Tasks Tab</strong> (filtered to 'My Tasks' by default)</p><h2>What QA Can Do</h2><h3>Full Access</h3><p><strong>ΓÇó Tasks</strong> - View, create, update, complete test-related tasks</p><p><strong>ΓÇó Sprints</strong> - Participate in sprint planning, view sprint board, update task status</p><p><strong>ΓÇó Timeline</strong> - View Gantt chart, see task dependencies, check deadlines</p><p><strong>ΓÇó Findings</strong> - Create bugs, update severity, assign to developers, verify fixes, close findings</p><p><strong>ΓÇó Time Tracking</strong> - Log time spent on testing activities</p><p><strong>ΓÇó Testing Access</strong> - Manage test environments, credentials, deployment links</p><p><strong>ΓÇó Recurring Tasks</strong> - Create and manage recurring test tasks (e.g., weekly regression tests)</p><p><strong>ΓÇó Files</strong> - Upload test reports, screenshots, logs</p><h3>Read-Only Access</h3><p><strong>ΓÇó Overview Dashboard</strong> - View project health, milestones, findings summary</p><p><strong>ΓÇó Milestones</strong> - See delivery deadlines and milestone status</p><p><strong>ΓÇó Activity Log</strong> - Track project changes and updates</p><h3>No Access</h3><p>ΓÇó Discussions - No access to project-level strategic discussions</p><p>ΓÇó Updates - Cannot post project status updates (PM responsibility)</p><p>ΓÇó Reports - Cannot generate executive reports</p><p>ΓÇó Team - Cannot view team capacity or manage team members</p><p>ΓÇó Financials - No access to budget, invoices, or financial data</p><p>ΓÇó Automations - Cannot configure project automation rules</p><p>ΓÇó Client Management - Cannot view or manage clients</p><h2>Project Visibility</h2><p>QA can only see projects where they are explicitly assigned as a team member. They do not see all company projects (unlike PM/OPS), client-facing project lists, or projects from other teams.</p><h2>Findings Workflow</h2><p>QA's primary function is managing the bug/issue lifecycle:</p><p>1. QA Creates Finding</p><p>2. Sets Severity (Critical/High/Medium/Low)</p><p>3. Assigns to Developer</p><p>4. Developer Fixes ΓåÆ Updates Status to 'Ready for Testing'</p><p>5. QA Verifies Fix</p><p>6. If Passed: QA Closes Finding</p><p>7. If Failed: QA Reopens with Comments</p><h1>4.
